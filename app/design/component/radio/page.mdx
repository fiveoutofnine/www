# Radio

A set of checkable buttons, where no more than one of the buttons can be checked at a time.

<Callout>
  Requires the `'use client'` directive if [React Server Components](https://react.dev/blog/2023/03/22/react-labs-what-we-have-been-working-on-march-2023#react-server-components) are being used.
</Callout>

## Type

There are two types of radio buttons: [`normal`](#normal) and [`detailed`](#detailed). When using `normal` radio buttons, `<Radio.Item />` must be supplied with a unique `id` prop, and an associated, descriptive `<label>` must be used with its `htmlFor` prop set to `id`. Conversely, when using `detailed` radio buttons, `id` must not be supplied. Instead, a descriptive `title` must be supplied, which will be used as the radio button's label and automatically associated with the radio button.

## Detailed

Unlike `normal` radio buttons, `detailed` radio buttons must have a descriptive `title` prop and may optionally have `descriptive`, `icon`, and `intent` props.

  );
